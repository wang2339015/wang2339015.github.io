jsp 基础知识

html代码：

input type=“password” name=“pw”
input type=“text” name=“txt”
textarea name=“txtarea”

取password 的值
String pw = request.getParameter(“pw”);

取 text 文本框的值
String pw = request.getParameter(“txt”);

取 textarea 文本域的值
String pw = request.getParameter(“txtarea”);

input type=“radio” name=“sex” value=“1”
input type=“radio” name=“sex” value=“0

取 radio 单选按钮的值
String result = request.getParameter(“sex”);

select name=“sel“
option value=“101”>选项1选项2

取下拉菜单 select 的值
String result = request.getParameter(“sel”);

页面跳转 重点 ：重定向 和 请求转发 一定要分清 .

重定向 ：
response.sendRedirect(“index.jsp”);
代参数 ：
response.sendRedirect(“index.jsp?aa=123”);
多参数 ：
response.sendRedirect(“index.jsp?aa=123&bb=456”);

请求转发：
创建 requestDispatcher类对象
requestDispatcher rd = request.getRequestDispatcher();
通过forward 方法跳转
rd.forward(request,response);

区别 ： 重定向跳转页面时，传到的页面不接受参数.
请求转发跳转页面时，传到的页接收参数。

请求转发时 携带Obeject
页面通过 request.setAttribute() 来存放数据 。
Obeject ob = new Object();
request.setAttribute(“myob”,ob);

页面通过 getAttribute 方法来 读取数据 。
Object result = request.getAttribute(“myob”);

session 对象

作用域 ： 当使用浏览器访问web服务器时，会建立一次会话（session）,我们可以往session里存储数据，在于服务器断开之前都是有效的。

使用方法： 往session 存储数据
session.setAttribute(“key”,”value”);
从session 获取数据
Object value = session.getAttribute(“key”);
注意： session 返回的Object 类型， 如果需要别的类型，请强转。

application 对象

作用域： applcation 是一个应用级对象 ， 在整个网站拥有唯一作用域 ， 如果存在多个session 那么多个session 公用一个application

使用方法 ： 向application 里存入数据
application.setAttribute(“key”,”value”);
从application 中获取数据
Object value = application.getAttribute(“key”);
注意： 返回值也是Object 跟session 一样 。

以上就是JSP最基础，也是最实用的知识了

Git 基本命令

hexo

npm install hexo -g #安装
npm update hexo -g #升级
hexo init #初始化
简写

hexo n “我的博客” == hexo new “我的博客” #新建文章
hexo p == hexo publish
hexo g == hexo generate#生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy#部署

服务器

hexo server #Hexo 会监视文件变动并自动更新，您无须重启服务器。
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP

hexo clean #清除缓存 网页正常情况下可以忽略此条命令
hexo g #生成静态网页
hexo d #开始部署

监视文件变动

hexo generate #使用 Hexo 生成静态文件快速而且简单
hexo generate –watch #监视文件变动

完成后部署

两个命令的作用是相同的
hexo generate –deploy
hexo deploy –generate
hexo deploy -g
hexo server -g

草稿

hexo publish [layout]

模版

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub

hexo new [layout] <br>hexo new photo “My Gallery”<br>hexo new “Hello World” –lang tw
